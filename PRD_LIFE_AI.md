## Решения, принятые 2026-07-17 (уточняют разделы ниже)

- **AI-провайдер:** Claude (Anthropic), не OpenAI. Уже есть рабочая интеграция (`claude_ai.py`), это основной инструмент проекта.
- **Голос (STT):** OpenAI Whisper API — только для распознавания речи, вызывается внутри кода. Личный чат с ботом остаётся приватным (без группового чата со сторонними ботами — отклонили вариант с `@smartspeech_sber_bot` из-за приватности данных о здоровье и лишнего усложнения UX).
- **Инфраструктура MVP 1:** SQLite вместо PostgreSQL + Redis. Проще и дешевле в хостинге для одного пользователя; слой БД скрыт за интерфейсом, апгрейд на Postgres возможен позже без переписывания бизнес-логики. Docker — опционально, не обязателен на старте.
- **Данные:** начинаем с чистого листа, историю из старого `fitness.db` не переносим.
- **Расположение проекта:** переиспользуем существующую папку `fitness-bot` (`Клод — Проекты\fitness-bot\`), код внутри переписывается под новую архитектуру.

# 1. Идея проекта

Создать Telegram-бота, который становится персональным AI-помощником пользователя.

Главная задача бота — не собирать данные ради данных, а помогать человеку понимать, почему он чувствует себя именно так, почему меняется вес, энергия, настроение и продуктивность.

Бот должен искать закономерности между:

* здоровьем;
* сном;
* работой;
* стрессом;
* тренировками;
* питанием;
* болезнями;
* событиями жизни.

Главная цель — не контроль пользователя, а помощь в принятии решений.

---

# 2. Главный принцип

Пользователь НЕ должен вести дневник.

AI ведёт дневник самостоятельно.

Пользователь просто общается с ботом естественным языком или голосом.

---

# 3. Основные принципы

### Минимум действий

Среднее время общения:

30–60 секунд в день.

Никаких длинных анкет.

---

### Свободный ввод

Пользователь может написать:

> Сегодня работала 10 часов, была йога, выпила бокал пива.

или

отправить голосовое.

AI самостоятельно понимает информацию.

---

### Никаких повторяющихся вопросов

Если сегодня уже записан вес —

бот больше не спрашивает вес.

Если уже записана тренировка —

не спрашивает тренировку.

---

### AI должен помнить контекст

Например:

> В прошлом месяце пользователь болел.

или

> Последние две недели работал по 11 часов.

Это должно использоваться при анализе.

# 7. Архитектура AI

AI состоит из нескольких независимых сервисов.

---

Получает текст.

Возвращает JSON.

Пример.

Вход

> Сегодня работала 9 часов и играла в бадминтон.

```json
{
  "work_hours":9,
  "badminton":120
}
```

Никаких комментариев.

Только JSON.

## Не входит в MVP

Пока **не реализовывать**:

* Apple Health.
* Google Fit.
* Garmin.
* Умные часы.
* Календарь.
* Финансы.
* Автоматическое планирование тренировок.
* Продвинутого AI-коуча с ежедневными рекомендациями.

---

## Последнее требование (самое важное)

Этот проект должен быть написан так, чтобы через год его можно было превратить в полноценного персонального AI-ассистента без переписывания архитектуры.

Все компоненты должны быть независимыми, с чёткими интерфейсами и возможностью заменить AI-провайдера (например, OpenAI на другой сервис) без изменения бизнес-логики.
