## Записи

### 2026-07-17 — AI-провайдер: Claude, а не OpenAI

- Контекст: нужен основной AI-движок для анализа и диалога с пользователем.
- Решение: Claude (Anthropic) — основной инструмент; уже есть рабочая интеграция (`claude_ai.py`).
- Почему так: интеграция уже работала на момент решения.
- Альтернативы: OpenAI — используется отдельно, только для распознавания речи (Whisper API), не для диалога.
- Когда пересмотреть: если качество ответов Claude перестанет устраивать для этого сценария.

### 2026-07-17 — Инфраструктура MVP: SQLite вместо PostgreSQL + Redis

- Контекст: нужна БД для персонального бота одного пользователя.
- Решение: SQLite на MVP-этапе, слой БД скрыт за интерфейсом.
- Почему так: проще и дешевле в хостинге для одного пользователя; апгрейд на Postgres возможен позже без переписывания бизнес-логики.
- Альтернативы: PostgreSQL + Redis — отклонены для MVP как избыточные.
- Когда пересмотреть: если появится больше одного пользователя или вырастут требования к параллельным записям.

### 2026-07-17 — История из старого fitness.db не переносится

- Контекст: у продукта было предыдущее воплощение (обычный фитнес-трекер) со своей БД.
- Решение: начинаем с чистого листа, старую историю не мигрируем.
- Почему так: LIFE AI — концептуально другой продукт (аналитика жизни, а не трекер), перенос старых данных не оправдан.
- Альтернативы: миграция истории — отклонена.
- Когда пересмотреть: не пересматривать, если только не понадобятся конкретные архивные данные.

### 2026-07-17 — Без группового чата со сторонними ботами

- Контекст: рассматривался вариант интеграции с `@smartspeech_sber_bot` для группового сценария.
- Решение: личный чат с ботом остаётся приватным, без группового чата со сторонними ботами.
- Почему так: приватность данных о здоровье + вариант усложнял UX без явной пользы.
- Альтернативы: групповой чат с `@smartspeech_sber_bot` — отклонён.
- Когда пересмотреть: если появится явный запрос на групповой/семейный сценарий с отдельным согласием на приватность.
