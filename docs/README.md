# Карта документации проекта LIFE AI

Этот файл — точка входа в документацию проекта. Он объясняет, где что лежит,
чтобы факты не размазывались по разным местам.

## Где что лежит

- **Правила работы с проектом** → [`../CLAUDE.md`](../CLAUDE.md) в корне репозитория.
- **Продуктовые требования** → [`../PRD_LIFE_AI.md`](../PRD_LIFE_AI.md) — что за продукт, принципы, решения по архитектуре (уже существовал в репозитории, не дублируется здесь).
- **Долговечная память проекта** → [`project-memory.md`](project-memory.md). Где живёт бот, деплой, важное про рассинхрон с сервером.
- **Известные баги и gotchas** → [`bugs-and-gotchas.md`](bugs-and-gotchas.md).
- **Журнал долговечных решений** → [`decision-log.md`](decision-log.md).

## Правило одного места

Каждый факт должен иметь одно каноническое место. Архитектурные решения продукта — в `PRD_LIFE_AI.md` (не копировать сюда). Операционные факты (VPS, деплой, баги) — в файлах этой папки.
